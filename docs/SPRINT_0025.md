# Sprint 0025 - Checklist Físico en Validación

## Objetivo

Permitir que el operador acredite en la etapa de validación que la documentación existe físicamente en el expediente, sin cargar todos los PDFs.

## Flujo

1. La validación detecta faltantes.
2. El operador abre el checklist físico.
3. Marca la documentación que obra en expediente.
4. El sistema registra el checklist en historial.
5. La validación se recalcula.
6. Si todas las evidencias quedan acreditadas, el expediente puede salir VALIDADO.

## Evidencias contempladas

- Facturas.
- Remito / Conformidad / Acta.
- CAE.
- ARCA.
- ARBA.
